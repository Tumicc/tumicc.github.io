    • 从 Java 开发到博客博主：我的技术转型之路

    🚀 背景介绍

    大家好，我是 Nolan，一名资深Java 开发工程师。在这个 Java 世界里，我写了数不清的 if-else、try-catch，以及无数个 public static void main 方法。但最近，我突然发现——我写的代码比文章多，注释比日记长！于是，我决定用 Hexo 搭建一个技术博客，记录我的踩坑日记、技术心得，以及偶尔的吐槽。

    💻 我的技术栈（Java 版）

    作为一名 Java 从业者，我的日常是这样的：

    • Java：从 Servlet 写到 Spring Boot，从 XML 配置到 Java 配置，从单体架构到微服务（虽然现在还在学习中）。
    • 框架：Spring/Spring Boot、MyBatis、Spring Cloud（目前还在努力掌握 Cloud 的分布式事务和网关）。
    • 数据库：MySQL（主库、从库、分库分表）、Redis（缓存、分布式锁、布隆过滤器）。
    • 工具：Maven、Gradle、Git、Jenkins（CI/CD 的入门选手）。
    • 其他：Linux（会写一些 Shell 脚本）、Docker（玩过几个容器）。

    总之：Java 是我的饭碗，但我的野心不止于此！

    😂 为什么突然想写博客？

    1. 总结技术经验：每次遇到问题，百度 一搜，发现“哦，原来是这个问题”，但下次遇到又忘了。所以，博客是用来记住自己的愚蠢的。
    2. 技术分享：Java 社区那么大，我也想贡献点微小的力量（虽然可能没人看）。
    3. 记录生活：Java 代码写的够多了，我也想写点有人味的文章（比如吐槽加班、记录成长）。

    🎯 博客计划

    • 技术专栏：Java、Spring、分布式、业务经验。
    • 生活随笔：偶尔谈谈人生（比如如何优雅地拖延症）。
    • 踩坑记录：记录那些让我头秃的问题和解决方案（比如 Nacos 配置中心挂了怎么办）。

    📝 今天的第一篇文章

    这是我的 Hexo 博客第一篇文章，主要是试试水，看看 Markdown 和 Hexo 能不能搞点花样（比如插入表情包 ✨）。如果看到这里，说明你还挺有耐心的，点个赞再走吧！

    如果你也是一名 Java 开发者，或者对技术感兴趣，欢迎在评论区交流！如果发现我写的有问题，欢迎友好地批评指正（毕竟我也还在学习中）。
